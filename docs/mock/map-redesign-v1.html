Recommendation: Option A (Service Cards) — It preserves the spatial relationship between services and dependencies (the "map" quality) while eliminating OTel tier internals. The horizontal card layout makes the 1–3 service case (our primary target audience) look intentional rather than empty. Dependencies are visually distinct via dashed borders and "External" tags. Connector arrows show which service depends on what.

Information Hierarchy

"What users can tell in 2 seconds":
① Which services exist  →  ② Which are broken (red dot + error %)  →  ③ Which dependency is the root cause (dashed card with red)  →  ④ Click red → incident detail

Stays on Map

  • Service name + overall status (rollup)
  • Route list with status dots
  • Error rate per route
  • Req/s per route
  • Dependency name + status
  • Service → dependency edges

Moves to Incident Detail

  • P95 latency per route
  • Span-level traces
  • Runtime units (internal spans)
  • Causal chain
  • Edge traffic counts
  • Badge details
What Gets Removed
Removed from the map:
→ 3 swim lanes (ENTRY POINTS / RUNTIME UNITS / DEPENDENCIES)
→ Tier labels and divider lines
→ Tier-based layout computation (TIER_Y, TIER_DIVIDERS)
→ Node shape differentiation by tier (square vs pill vs dashed)
→ 7-item legend → simplified to 4 items (healthy / degraded / critical / external dep)
→ runtime_unit as a visible map element (metrics roll up into service)

Preserved:
→ Click red element → zoom to incident (same incidentId click behavior)
→ Status dot colors (green/amber/red)
→ Animated traffic dots on edges (adapted for service→dep connectors)
→ Design tokens, font stack, border styles
→ Stats bar and incident strip below the map
Click Targets & Drill-Down
Clickable elements:
→ Route row (if it has an incidentId): click → zoom to incident board (level 1)
→ Service card header (if any route has incident): click → zoom to first matching incident
→ Dependency card (if it has an incidentId): click → zoom to incident board

Non-clickable: Healthy routes with no incident — cursor remains default.

Visual feedback: cursor: pointer + hover highlight on clickable elements. Focus ring (outline: 2px solid var(--teal)) for keyboard navigation.
