INC-0892Critical

Stripe API rate limit cascade causing payment failures across all checkout flows

68% checkout errorsStripe 429payment-serviceorder-service

Immediate Action

Enable request batching on StripeClient and add exponential backoff with jitter
Stripe is returning 429 because payment-service sends one API call per transaction with no batching. During the current 3x traffic surge, this exceeded Stripe's 100 req/sec rate limit. Batching + backoff will reduce call volume by ~80%.
Do not: Request a Stripe rate limit increase — this masks the underlying design flaw.
