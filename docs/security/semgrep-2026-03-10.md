## Findings

### [HIGH] GitHub Actions — Shell Injection（2件）

**Rule:** `run-shell-injection`
**File:** `.github/workflows/diagnose.yml`

#### Finding 1 — Line 42（`curl` ステップ）

```yaml
# 現状（危険）
run: |
  curl ... "${{ inputs.packet_id }}"
```

`inputs.packet_id` がシェル展開前にそのまま埋め込まれる。ReceiverはOTelデータから生成したIDをdispatch時に渡す（ADR 0021）。悪意あるOTelエージェントが `;`・`$()`・`&&` を含むIDを送ると、GitHub Actionsランナーで任意コマンドが実行され、`ANTHROPIC_API_KEY` 等のシークレットが漏洩する。

**攻撃経路:** 悪意あるOTelエージェント → packet_id に shell metachar → Receiver dispatch → runner でコマンド実行

**Fix:**
```yaml
env:
  PACKET_ID: ${{ inputs.packet_id }}
run: |
  # 入力フォーマット検証
  if ! [[ "$PACKET_ID" =~ ^[a-zA-Z0-9_-]+$ ]]; then
    echo "Invalid packet_id format" && exit 1
  fi
  curl ... "$PACKET_ID"
```

#### Finding 2 — Line 51（`node` CLI ステップ）

同根因。`${{ inputs.incident_id }}` が `--callback-url` 引数のシェル文字列内に展開される。

**Fix:** 同様に `env:` ブロックへ移動 + 正規表現バリデーション。

### [MEDIUM] Docker — root実行（3件）

**Rule:** `missing-user`
**対象:** validation配下でホストポートをbindするコンテナ

| File | 理由 |
|------|------|
| `validation/apps/web/Dockerfile:11` | DBクレデンシャルあり・外部トラフィック受信 |
| `validation/apps/mock-stripe/Dockerfile:7` | ホストポートbind |
| `validation/tools/loadgen/Dockerfile:7` | `/__admin` 無認証APIあり |

ベースイメージ `node:20-alpine` は `node` ユーザーを持つが、明示的な `USER` 命令がなく root で実行される。コンテナ侵害時のエスカレーションリスクが上がる。

**Fix（全Dockerfile共通）:**
```dockerfile
# COPYの後、CMDの前に追加
RUN chown -R node:node /app
USER node
```

## TS/JS クリーン — 良好な設計の証左

0件は偶然ではない。以下が機能している：

### トリアージ根拠

**FALSE_POSITIVE（6件）— validation/ コードのみ**

- `js/resource-exhaustion`（3件）: validation mock serversのタイマー。攻撃者がアクセスできるエンドポイントはローカルDocker内に限定され、インターネット非公開。production影響なし。
- `js/file-system-race`（1件）: scenario-runner/run.jsのファイルチェック。CI/CDパイプライン専用ツールで攻撃者が制御できる入力経路なし。
- `js/remote-property-injection`（2件）: `...headers` スプレッドと `profiles[body.profile]` の書き込み。mock-stripeとloadgenはシナリオ制御専用の内部APIで、外部公開なし。

**TRUE_POSITIVE（2件）— テストコード**

- `js/insecure-temporary-file`（2件）: `e2e/global-setup.ts` と `cli.test.ts` でOSのtmpディレクトリに予測可能なファイル名で書き込み。シンボリックリンク攻撃（symlink attack）のリスク。ただしテスト環境限定で本番影響なし。
